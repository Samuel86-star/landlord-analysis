# Markdown 书写规范

> 本文档定义项目中所有 Markdown 文件的书写规范，确保文档格式统一、lint 无警告。

---

## 一、代码块规范

### 1.1 必须指定语言类型

所有代码块必须指定语言类型，不得使用空的 ` ``` ` 标记。

**错误示例：**

```text
```
SELECT * FROM table;
```
```

**正确示例：**

```sql
SELECT * FROM table;
```

### 1.2 常用语言类型

| 语言类型 | 标记 | 适用场景 |
| -------- | ---- | -------- |
| SQL | `sql` | 数据库查询语句 |
| Bash | `bash` | Shell 命令、脚本 |
| Python | `python` | Python 代码 |
| JavaScript | `javascript` 或 `js` | JS 代码 |
| JSON | `json` | JSON 数据 |
| YAML | `yaml` | 配置文件 |
| Text | `text` | 纯文本、无语法高亮需求 |
| Markdown | `markdown` | Markdown 示例 |

---

## 二、表格规范

### 2.1 分隔符格式

表格分隔行中，管道符 `|` 与内容之间必须有空格。

**错误示例：**

```text
| 字段 | 类型 | 说明 |
|--|----|------|
| id | int | 主键 |
```

**正确示例：**

```text
| 字段 | 类型 | 说明 |
| ---- | ---- | ---- |
| id | int | 主键 |
```

### 2.2 列对齐格式

分隔符中的对齐标记（`:-`, `-:`, `:-:`）与管道符之间也需有空格：

```text
| 左对齐 | 居中 | 右对齐 |
| :----- | :---: | -----: |
| 内容 | 内容 | 内容 |
```

### 2.3 表格前后空行

表格前后必须有空行与其他内容分隔。

---

## 三、标题规范

### 3.1 标题层级递进

标题层级应逐级递进，不得跳级：

- 一级标题 `#`：文档标题（仅文档开头使用一次）
- 二级标题 `##`：章节标题
- 三级标题 `###`：小节标题
- 四级标题 `####`：更细粒度（谨慎使用）

**错误示例：**

```text
# 文档标题
#### 直接跳到四级标题
```

**正确示例：**

```text
# 文档标题

## 第一章

### 1.1 小节
```

### 3.2 避免重复标题

同一文档内，同级标题内容不得完全相同。如果确实需要相同结构，应添加差异化描述。

**错误示例：**

```text
## 查询一

### 说明

### 参数说明

## 查询二

### 说明

### 参数说明
```

**正确示例：**

```text
## 查询一

### 查询逻辑说明

### 可调参数列表

## 查询二

### 统计逻辑说明

### 可调参数配置
```

### 3.3 标题前后空行

标题前后必须有空行（文档开头的一级标题除外）。

---

## 四、列表规范

### 4.1 列表符号统一

使用 `-` 作为无序列表符号，不使用 `*` 或 `+`。

**正确示例：**

```text
- 项目一
- 项目二
  - 子项目
```

### 4.2 列表缩进

子列表缩进 2 个空格（或 4 个空格，全文统一）。

### 4.3 列表前后空行

列表前后必须有空行与其他内容分隔。

---

## 五、链接规范

### 5.1 链接片段命名

内部链接片段（锚点）必须有效且唯一。GitHub 自动生成的片段基于标题文本（小写、空格替换为连字符）。

**链接到标题示例：**

```text
## SQL 查询语句

参见 [查询说明](#sql-查询语句)
```

### 5.2 避免裸 URL

避免直接使用裸 URL，应使用链接语法：

**错误示例：**

```text
参见 https://example.com
```

**正确示例：**

```text
参见 [示例网站](https://example.com)
```

---

## 六、文件结尾规范

### 6.1 文件末尾换行

每个 Markdown 文件末尾必须有且仅有一个空行。

---

## 七、其他规范

### 7.1 行尾空格

避免行尾多余空格（部分编辑器会自动添加，需注意清理）。

### 7.2 连续空行

避免连续多个空行，最多保留一个空行分隔内容块。

### 7.3 强调格式

使用 `**粗体**` 和 `*斜体*`，不使用 `__粗体__` 和 `_斜体_`。

---

## 八、规范检查清单

在创建或修改 Markdown 文件时，请确认以下事项：

| 检查项 | 要求 |
| ------ | ---- |
| 代码块语言类型 | 所有 ` ``` ` 后必须指定语言（如 `sql`） |
| 表格分隔符 | `| ---- |` 格式，管道符与内容间有空格 |
| 标题层级 | 逐级递进，不跳级 |
| 标题重复 | 同级标题内容不同 |
| 标题前后空行 | 标题前后有空行 |
| 列表符号 | 统一使用 `-` |
| 列表缩进 | 子列表缩进一致（2或4空格） |
| 链接片段 | 链接锚点有效且唯一 |
| 文件末尾 | 末尾有且仅有一个空行 |
| 行尾空格 | 无多余行尾空格 |
| 连续空行 | 无连续多个空行 |

## 九、常见 Lint 规则对照

| 规则编号 | 规则名称 | 说明 |
| -------- | -------- | ---- |
| MD018 | 代码块缺少语言标识 | ` ``` ` 后必须指定语言 |
| MD024 | 不允许重复标题 | 同级标题内容不得相同 |
| MD040 | 代码块应指定语言 | 同 MD018 |
| MD047 | 文件末尾换行 | 文件末尾应有空行 |
| MD051 | 链接片段无效 | 链接锚点必须存在 |
| MD055 | 表格风格 | 表格管道符格式 |
| MD056 | 表格列风格 | 表格列对齐格式 |
| MD060 | 表格列对齐风格 | 分隔符格式 |
