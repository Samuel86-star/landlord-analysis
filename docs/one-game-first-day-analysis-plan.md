# 低局数新用户流失分析 — 综合解析方案

> **核心目的**：深层次理解「为什么新用户仅完成少量对局（≤1局、≤2局、≤5局）后就流失」，系统化拆解流失的触发机制、高危人群特征、流失时间节点，并识别可干预的产品与运营杠杆。  
> **分析范围**：不仅限于「1局」的极端情况，扩展到「2-5局」这个最大流失带，从更宏观的「局数-留存」曲线找规律。  
> **关联材料**：[`report/retention-global-report.md`](../report/retention-global-report.md)（全局层）、[`docs/retention-global.md`](retention-global.md)（取数口径与 SQL 体系）。

---

## 一、问题背景与已有结论

### 1.1 现状数据

[`retention-global-report.md`](../report/retention-global-report.md) 在 **3.2 首日对局数** 中给出的关键事实（口径为**登录留存**）：

| 分组 | 占全体新增比例 | 规模 | 次留率 | 7留率 | 核心特征 |
|------|---|---|---|---|---|
| **0 局** | 13.2% | ~11K | 15.56% | — | 未进入对局或异常流失 |
| **1 局** | **9.2%** | ~8K | **9.57%** | — | **全体最低**；完整体验后秒走 |
| **2-5 局** | **25.2%** | ~22K | **16.35%** | — | **最大流失带**；试验后放弃 |
| **6-10 局** | — | — | — | — | 存在度提升 |
| **11+ 局** | — | — | — | — | 高投入用户 |

### 1.2 关键观察

1. **1-5局用户占全体新增 34.4%**，是最大的改善空间
2. **负相关曲线陡峭**：1局最差（9.57%）→ 2-5局略好（16.35%）但仍低于平均
3. **2-5局是关键分水岭**：这个群体"有尝试、有投入"但最终放弃，特征复杂
4. **未被解释的问题**：这部分用户为什么在注册当日就停止对局？是体验差、是任务完成、是技术卡顿、还是选择？

## 二、分析目标与成功标准

### 2.1 分析目标

1. **精细化分层**：拆解 0/1/2-5/6-10/11+ 各分组的用户构成，找到差异的本质
2. **流失触发点识别**：在 1-5 局的用户中，定位他们在「哪一局之后」放弃的，以及那一局的特征
3. **高危人群特征**：跨越「渠道×首局结果×倍数×经济状态×时长×逃跑」的多维交叉，找到高流失签名
4. **流失时间节点**：首局完成距离注册多久、每局间隔多久、末局在当日几点，推断流失机制
5. **可干预杠杆排序**：Top 3-5 个产品、运营或渠道侧的干预点，每条附「预期收益」和「难度」评分

### 2.2 成功标准

- 能解释 60-80% 的低局数流失用户归因到具体的「流失类型」
- 给出 Top 3 可干预杠杆及其预期影响量级（如"优化首局配牌可降低1局占比 2-3 个百分点"）
- 与主报告结论一致或明确说明修正理由

## 三、核心指标与口径统一

### 3.1 「局数分组」定义

**首日对局数** = 同一 uid 在 reg_date 当天、斗地主 APP 真人对局（`robot != 1`）、银子房间（`play_mode IN (1,2,3,5)`）下的去重 `resultguid` 计数。

分组标准：
- **低局数**：≤5 局（重点分析）
- **极限低**：≤1 局
- **较低**：2-5 局（占比最大，改善空间最大）
- **适度**：6-10 局（形成习惯的临界点）
- **高投入**：11+ 局

### 3.2 建议并行汇报的三套「占比」

| 指标 | 分母 | 用途 |
|------|------|------|
| **A. 全体新增中低局占比** | 当日新增 APP 用户数 | 与报告对齐，看大盘 |
| **B. 有对局用户中低局占比** | 当日新增且当日对局≥1 的用户 | 回答「进来打牌的人里，多少人早早退出」 |
| **C. 经典玩法内低局占比** | 当日新增且当日在经典房间有对局 | 与新手默认路径对齐 |

### 3.3 流失时间节点指标

为了理解「用户在什么时候放弃」，需要计算：

| 指标 | 含义 | 数据源 |
|------|------|------|
| **首局耗时** | 注册后到首局结束经历的时间 | `first_game_time_unix - reg_datetime` |
| **局间间隔** | 首局到末局的总耗时 | `last_game_time_unix - first_game_time_unix` |
| **末局时刻** | 末局在当日的几点 | `HOUR(last_game_time_unix)` |
| **当日活跃区间** | 首局到末局跨越的时长 | `last_game_time_unix - first_game_time_unix` |

## 四、流失特征识别框架

### 4.1 低局数用户的六类流失签名

根据数据可观测线索，将低局数用户粗分为以下几类（可重叠）：

| 编号 | 流失类型 | 数据特征 | 可观测指标 |
|------|---------|---------|----------|
| **T1** | **首局体验极差** | 首局结果不好/时长异常/经济崩溃 | 首局负、首局胜率 <30%、首局时长 <60s 或 >10min、单局亏超 5000 |
| **T2** | **高倍冲击型** | 首几局倍数过高，银子波动剧烈 | 首局平均倍数 >24、单局波动幅度超过初始银子 50% |
| **T3** | **连败挫折型** | 早期连败导致心理崩溃 | 前 2-3 局连败、连败后无再进局 |
| **T4** | **逃跑异常型** | 逃跑行为异常多或首局即逃 | 首日逃跑率 >50%、首 2 局内已有逃跑 |
| **T5** | **破产触发型** | 经济崩溃到无法继续 | 货币谷值 <500、单局巨亏导致 end_money 不足房间门槛 |
| **T6** | **时间卡点型** | 注册-首局间隔长、末局很晚 | 首局耗时 >30min、末局时刻 >22:00、首末局间隔 <5min（秒开秒关） |

### 4.2 流失时间节点模型

将低局数用户按"在第几局后放弃"分层：

```
用户 A：注册 → 首局失败 → 放弃（1 局）
  特征：首局体验最差，通常归类为 T1

用户 B：注册 → 第 1 局胜 → 第 2 局大亏 → 放弃（2 局）
  特征：初期体验还可以，但某一局的打击过大，归类为 T2/T5

用户 C：注册 → 局 1-3 逐局平稳 → 第 4 局逃跑 → 放弃（4 局）
  特征：逃跑成为临界点，归类为 T4

用户 D：注册 → 对局 1-5 都打完 → 末局时间很晚且输钱 → 当日无再进
  特征：时间卡点 + 经济负面体验的复合
```

## 五、分析维度清单与产出物

### 5.1 必选维度（第一期）

| 维度 | 细分 | 产出 SQL / 表 | 优先级 |
|------|------|----------|------|
| **局数分布** | 0/1/2-5/6-10/11+ | 各分组占比、趋势、日周内分布 | P0 |
| **首局特征** | 结果、时长、倍数、经济变化 | 1-5 局 vs 6+ 局的首局对比（胜率、倍数、输赢差） | P0 |
| **末局特征** | 末局胜负、时刻、是否逃跑 | 末局在当日几点、末局是赢是输与放弃的关系 | P0 |
| **渠道** | 主要渠道分类 | 各渠道的低局占比、是否某渠道异常聚集 1 局用户 | P0 |
| **端/版本** | Android / iOS、客户端版本 | 不同客户端的低局占比与特征差异 | P0 |
| **连胜连败** | 前 N 局连败长度 | 1-5 局用户是否普遍在连败中放弃 | P1 |
| **经济曲线** | 银子峰谷值、破产标记 | 1-5 局用户的破产率、平均经济最差点 | P1 |
| **逃跑行为** | 逃跑次数、逃跑率 | 1-5 局用户的逃跑率 vs 高局用户对比 | P1 |
| **时间节点** | 首局耗时、局间间隔、末局时刻 | 推断时间压力或行为急切程度 | P2 |
| **玩法偏好** | 经典 / 不洗牌 / 赖子 | 首日玩法分布是否与局数相关 | P2 |

### 5.2 交叉分析（深化因果）

| 交叉维度 | 目标问题 |
|---------|---------|
| **首局胜负 × 局数分组** | 首局负的用户是否全部 ≤5 局？首局胜的用户是否更易继续 |
| **倍数水平 × 局数分组** | 高倍首局的用户是否更易流失 |
| **渠道 × 局数分组** | 是否某渠道（如特定买量广告）特别聚集低局用户 |
| **连败 × 是否逃跑 × 局数** | 连败+逃跑的用户是否特别集中在 ≤5 局 |
| **破产 × 末局时刻 × 局数** | 破产用户中，末局很晚的是否更易离开 |
| **首末局间隔 × 局数** | 打对局很快（间隔短）和很慢的用户，局数分布是否差异大 |

### 5.3 建议交付物

1. **一页 Executive Summary**
   - 低局数占比 A/B/C 与趋势
   - 最大贡献流失类型（T1-T6 各占比）
   - Top 3 可干预杠杆与预期收益

2. **分析数据表** 
   - `日期 × 局数分组 × 次留率` 透视表
   - `流失类型 × 占比` 分布表
   - `渠道 × 低局占比` 排序表

3. **可视化仪表板**（可选）
   - 局数分布直方图与留存曲线
   - 首末局特征对比雷达图
   - 流失类型占比堆积图

4. **可复跑 SQL 清单**（所有分析对应的 SQL）

## 六、关键假设与验证方式

### 6.1 核心假设

| 假设 | 预期 | 验证方法 |
|------|------|---------|
| **H1：首局失败是 1 局用户主要流失原因** | 1 局用户首局负占比 >70%；vs 多局用户 <50% | 数据交叉验证 |
| **H2：高倍局冲击导致 2-5 局用户放弃** | 2-5 局用户中首局倍数 >24 占比 >40% | 倍数分层对比 |
| **H3：破产是强烈流失信号** | 破产用户的次留仅为未破产用户 50% | 破产标记分析 |
| **H4：逃跑行为与流失强相关** | 有逃跑行为的用户平均局数 <5；无逃跑 >8 | 逃跑标记分析 |
| **H5：时间卡点（首局很晚）预示放弃** | 末局 >22:00 的用户中，当日无再进次日登录率 <5% | 时间节点分析 |
| **H6：某渠道存在「任务流」用户（一局走）** | 某渠道 1 局占比 >15%，其他渠道 <10% | 渠道维度聚集分析 |

### 6.2 无法靠对局表直接回答的问题

| 缺口 | 建议补强方式 |
|------|----------|
| 用户首 N 局后是否"主动放弃"还是"网络卡掉" | 客户端崩溃/连接日志 |
| 末局后用户点了什么（是否尝试再开一局但匹配超时） | 客户端关键路径埋点（"尝试开局"→"匹配中"→"匹配超时") |
| 是否渠道任务完成（"打满 1 局即可领奖励"）| 投放侧 Campaign ID 与 ODM 回调对齐 |
| 用户主观感受（觉得难、觉得无聊、觉得坑钱） | 调研问卷、评价文本分析 |

## 七、数据与 SQL 产出提纲

### 7.1 基础表：局数分组分布

```sql
-- 统计新增用户按首日对局数分层的分布 + 次留率
-- 分母 A：全体新增用户
-- 预期输出：日期、局数分组、用户数、占比、次留率
```

### 7.2 首局特征对比表

```sql
-- 1-5 局用户 vs 6+ 局用户的首局对比
-- 输出：首局胜率、首局倍数、首局时长、首局经济变化
```

### 7.3 末局特征对比表

```sql
-- 1-5 局用户的末局特征
-- 输出：末局结果、末局时刻、末局是否逃跑、末局经济变化
```

### 7.4 流失类型分桶表

```sql
-- 将 1-5 局用户按 T1-T6 分类，计算各类占比
-- 需要多层 CASE WHEN 判断：
--   T1：首局负 AND 首局胜率 <30%
--   T2：首局倍数 >24 OR 单局波动 >50%
--   T3：连败 >=2 局
--   T4：逃跑次数 >0 AND 逃跑率 >30%
--   T5：货币谷值 <500
--   T6：首局耗时长 OR 末局时刻晚
```

### 7.5 渠道 × 低局占比排序表

```sql
-- 按渠道汇总 1-5 局占比，找到聚集渠道
-- 输出：渠道、1 局占比、2-5 局占比、6-10 占比、11+ 占比
```

### 7.6 关键交叉表

```sql
-- 首局胜负 × 局数分组 → 各分组的首局胜率
-- 连败 × 局数分组 → 各分组的连败长度分布
-- 破产 × 局数分组 → 各分组的破产率、破产时间
```

## 八、行动方案与优先级

### 8.1 Top 3 可干预杠杆

根据数据分析结果，预期的干预优先级：

| 优先级 | 干预方向 | 具体措施 | 预期影响 | 难度 |
|------|---------|---------|---------|------|
| **P0** | 首局体验优化 | 前 3 局配牌保障胜率 ≥40% | 降低 1 局占比 2-3pp | 中 |
| **P0** | 连败保护 | 连败 2 局后触发弱AI对手 | 提升 2-5 局用户留存 3-5pp | 中 |
| **P1** | 倍数限制 | 前 5 局理论倍数 cap ≤12 | 防止银子剧烈波动 | 高 |
| **P1** | 破产兜底 | 破产后自动赠送 2000 银子 | 提升破产用户次留 5-10pp | 低 |
| **P1** | 时间引导 | 末局后引导"还要再来一局吗" | 转化 5-10% 的末局用户为多一局 | 低 |
| **P2** | 渠道策略 | 识别&隔离「一局走」渠道，调整投放 | 压降渠道侧低质流量 | 中 |

### 8.2 实验设计样板

以"首局配牌保障"为例：

| 维度 | 方案 |
|------|------|
| **实验假设** | 保障首 3 局胜率 ≥40% 的新用户的次留率比对照组高 15%+ |
| **实验分组** | 对照组（现有配牌）vs 实验组（新手胜率保障） |
| **样本量** | ~5000 新用户/组，7 天实验期 |
| **核心指标** | 次留率、1 局占比、2-5 局占比 |
| **副指标** | 首日胜率、首日倍数分布（是否有副作用） |
| **启动条件** | 数据分析发现首局负占比在 1 局用户中 >60% |
